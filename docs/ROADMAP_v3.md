## 1. 愿景与战略

### 1.1 三方融合架构

```
┌─────────────────────────────────────────────────────┐
│                  伪春菜 终局架构                      │
│                                                       │
│   ┌───────────┐  ┌───────────────┐  ┌─────────────┐ │
│   │ 伪春菜     │  │ AIRI          │  │ OpenClaw    │ │
│   │ 灵魂经济   │  │ 身体视觉      │  │ 自主大脑    │ │
│   │            │  │               │  │             │ │
│   │ 信赖等级   │  │ Live2D渲染    │  │ Agent人格   │ │
│   │ 钱包/充值  │  │ 表情驱动      │  │ 自主决策    │ │
│   │ 送礼/零花钱│  │ 服装系统      │  │ 记忆系统    │ │
│   │ 惊喜系统   │  │ TTS语音       │  │ Cron行为    │ │
│   └─────┬─────┘  └───────┬───────┘  └──────┬──────┘ │
│         │                │                  │        │
│         └────────────────┼──────────────────┘        │
│                          │                           │
│              ┌───────────┴───────────┐               │
│              │  Electron / Web / iOS  │               │
│              │       三端统一交付      │               │
│              └───────────────────────┘               │
└─────────────────────────────────────────────────────┘
```

- **伪春菜（灵魂经济）**：信赖等级、钱包充值、送礼零花钱、惊喜系统 —— 完整的虚拟经济闭环
- **AIRI（身体视觉）**：Live2D角色渲染、情感表情驱动、服装外观系统、TTS语音合成
- **OpenClaw（自主大脑）**：Agent人格与自主对话、Cron定时行为、记忆整理与召回、决策引擎

### 1.2 核心差异化

灵魂经济系统是本项目区别于所有现有伪春菜产品的核心壁垒：

| 维度 | 竞品现状 | 我们的方案 |
|------|---------|-----------|
| 情感反馈 | 纯文字/表情 | 信赖等级量化 + Live2D情感驱动 |
| 商业模式 | 订阅/按次付费 | 送礼经济 + 零花钱分成（60/40） |
| 惊喜互动 | 无 | 确定性触发惊喜 + O2O实体配送 |
| 角色成长 | 无/简单升级 | 16000信赖满级 + 外观逐步解锁 |
| Agent智能 | 单一LLM调用 | OpenClaw自主大脑 + 记忆系统 |

### 1.3 终局愿景

Live2D角色（AIRI渲染引擎） + OpenClaw自主大脑（NAS本地部署） + 完整经济闭环（充值→送礼→零花钱→惊喜→O2O），三端（Electron桌面端 / Web PWA / iOS）统一交付，中国市场本地化（微信登录 + 微信支付/支付宝 + 中文语音）。

## 附录: 蜂群执行规范

### 并行Agent协作原则

1. **接口先行**: 每轮开始前，先定义Agent间的接口契约（类型/API/事件）
2. **独立可测**: 每个Agent的交付物可独立运行测试，不依赖其他Agent的产出
3. **合并验证**: 所有Agent完成后，执行集成测试验证交互正确性
4. **冲突解决**: 文件冲突时以接口契约为准，由最后的集成验证Agent统一调整

### 质量门禁流程

```
Agent完成 → 单元测试 → 类型检查 → 集成合并 → 集成测试 → 门禁检查 → 通过/打回
```

- 门禁检查不通过 → 定位问题Agent → 修复 → 重新验证
- 连续2次门禁不通过 → 暂停，重新评估方案
